# 运维与故障处理

## 1. 日常检查

每日检查：

- 行情同步是否正常；
- WebSocket是否连接；
- 数据库备份是否成功；
- 系统事件是否有ERROR/CRITICAL；
- 未完成订单是否异常；
- Kill Switch状态是否符合预期。

---

## 2. 故障等级

| 等级 | 说明 | 处理 |
|---|---|---|
| INFO | 普通事件 | 记录 |
| WARNING | 非阻断异常 | 提醒用户 |
| ERROR | 功能异常 | 暂停相关模块 |
| CRITICAL | 交易安全风险 | 触发Kill Switch |

---

## 3. 常见故障

### 3.1 WebSocket断开

处理：

1. 标记连接异常；
2. 禁止新单；
3. 启动REST轮询；
4. 重连；
5. 恢复后对账。

### 3.2 下单超时

处理：

1. 不立即重试；
2. 根据clientOid查询订单；
3. 如果存在则同步状态；
4. 如果不存在再判断是否允许重新提交。

### 3.3 止损设置失败

处理：

1. 触发CRITICAL；
2. 开启Kill Switch；
3. 提醒用户人工检查；
4. 提供补设止损确认操作。

### 3.4 数据库写入失败

处理：

1. 禁止执行；
2. 保留内存错误上下文；
3. 尝试恢复连接；
4. 恢复后补写事件。

---

## 4. 对账机制

系统定期执行：

- 本地订单 vs 交易所订单；
- 本地持仓 vs 交易所持仓；
- 本地成交 vs 交易所成交；
- 本地止损止盈 vs 交易所计划单。

发现不一致：

- 标记RECONCILIATION_REQUIRED；
- 禁止新单；
- 提醒用户。

---

## 5. 升级流程

1. 备份数据库；
2. 关闭Execution；
3. 部署新版本；
4. 执行迁移；
5. 运行健康检查；
6. 先开启只读；
7. 确认正常后再开启Execution。
