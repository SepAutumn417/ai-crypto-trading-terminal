# ADR-0001：选择L4而不是L5

## 状态

Accepted

## 背景

系统需要提升自动化程度，减少用户手动在交易平台填写订单参数的时间和错误。

## 决策

系统v1.0目标设为 L4：用户确认后执行。

系统不做L5全自动交易。

## 原因

- L4能提高执行效率；
- L4保留用户最终确认；
- L4更容易控制风险；
- L5需要更复杂的策略验证和风控体系；
- 当前阶段系统目标是个人交易终端，不是全自动机器人。

## 影响

- 需要Execution Engine；
- 需要Order Preview；
- 需要用户确认流程；
- 需要订单状态机；
- 需要Kill Switch。
