# MVP验收标准

## v0.1 验收：交易计划与仓位计算

必须满足：

- 可以创建交易计划；
- 可以输入入场价、止损价、止盈价；
- 可以计算风险金额；
- 可以计算止损距离；
- 可以计算名义仓位；
- 可以计算保证金；
- 可以计算盈亏比；
- 可以输出风控结论；
- 可以保存计划；
- 单元测试覆盖核心计算。

不通过条件：

- 仓位计算错误；
- 无止损仍可通过；
- 超杠杆仍可通过；
- 风控结果未保存。

---

## v0.2 验收：行情与图表

必须满足：

- 可以获取至少 BTCUSDT K线；
- 可以存储K线；
- 可以展示K线图；
- 可以切换周期；
- 数据重复同步不会重复写入。

---

## v0.3 验收：市场结构识别

必须满足：

- 能识别Swing High/Low；
- 能输出趋势/震荡状态；
- 能标记支撑压力；
- 能生成结构快照；
- 能在图表中展示关键位。

---

## v0.4 验收：自动候选计划

必须满足：

- 系统能生成候选计划；
- 候选计划有状态机；
- 候选计划有入场区、止损、目标；
- 候选计划能通过风控初筛；
- Opportunity Radar 可展示。

---

## v0.5 验收：AI评估

必须满足：

- AI输入结构化；
- AI输出JSON；
- AI不能覆盖风控；
- AI能解释机会质量；
- AI能解释风险；
- AI输出保存到数据库。

---

## v0.6 验收：订单预览与Dry Run

必须满足：

- 可以生成Order Intent；
- 可以生成订单预览；
- 可以生成交易所请求体；
- 可以执行Dry Run；
- 不会真实提交订单；
- clientOid唯一；
- 执行日志完整。

---

## v0.7 验收：只读同步

必须满足：

- 可以读取账户权益；
- 可以读取持仓；
- 可以读取订单；
- API Key不需要交易权限；
- 同步失败时有错误提示。

---

## v0.8 验收：小额L4执行

必须满足：

- 必须用户确认；
- 必须风控通过；
- 必须止损；
- 必须逐仓；
- 可以提交限价单；
- 可以同步订单状态；
- 可以设置或确认止盈止损；
- Kill Switch有效；
- 异常时禁止新单。

---

## v1.0 验收：完整L4终端

必须满足：

- 自动机会雷达可用；
- 自动计划生成可用；
- 动态仓位计算可用；
- 风控引擎可用；
- AI评估可用；
- 用户确认执行可用；
- 订单监控可用；
- 交易日志可用；
- 复盘统计可用；
- 系统安全机制可用。
