# 产品需求文档 PRD

## 1. 产品名称

**AI Personal Trading Terminal L4**  
中文名：**AI个人交易终端 L4**

---

## 2. 产品定位

一个面向个人加密货币合约交易者的半自动交易工作台。

系统通过规则算法、AI评估和交易所API集成，帮助用户完成从市场分析到用户确认执行的完整交易流程。

一句话定位：

> 基于算法风控与AI评估的个人半自动交易终端。

---

## 3. 核心价值

### 3.1 减少人工计算错误

系统自动根据账户权益、风险比例、入场价、止损价、杠杆、交易所精度规则计算：

- 风险金额；
- 止损距离；
- 最大名义仓位；
- 合约数量；
- 所需保证金；
- 预估手续费；
- 盈亏比。

### 3.2 减少临场情绪干扰

系统在下单前强制进行：

- 风控检查；
- 机会评级；
- 订单预览；
- 用户确认；
- 纪律状态检查。

### 3.3 提升交易执行效率

系统自动生成交易所订单参数，避免用户在交易平台中重复填写：

- 方向；
- 价格；
- 数量；
- 杠杆；
- 止损；
- 止盈；
- 保证金模式。

### 3.4 形成复盘闭环

系统自动记录：

- 计划；
- 风控结果；
- AI评估；
- 用户确认；
- 下单请求；
- 交易所响应；
- 成交记录；
- 持仓变化；
- 平仓结果；
- 复盘标签。

---

## 4. 用户角色

### 4.1 个人交易者

核心用户，使用系统进行个人交易辅助。

需求：

- 快速了解市场结构；
- 自动生成候选计划；
- 自动计算仓位；
- 快速完成下单；
- 控制风险；
- 复盘交易表现。

### 4.2 系统管理员

本项目v1.0为单用户系统，管理员通常也是用户本人。

职责：

- 管理API Key；
- 管理风控配置；
- 开启/关闭执行模式；
- 查看系统日志；
- 处理异常。

---

## 5. 关键业务流程

## 5.1 自动机会发现流程

```text
行情同步 → 市场结构识别 → 关键位识别 → 候选计划生成 → 机会评级 → 风控初筛 → AI解释 → 机会雷达展示
```

## 5.2 手动计划检查流程

```text
用户填写计划 → 仓位计算 → 风控检查 → AI解释 → 保存计划 / 放弃计划 / 进入订单预览
```

## 5.3 L4确认执行流程

```text
计划通过风控 → 生成订单预览 → 用户二次确认 → Execution Engine提交订单 → 同步订单状态 → 设置/确认TP SL → 进入持仓监控 → 自动记录日志
```

## 5.4 复盘流程

```text
订单关闭 → 生成交易记录 → 计算R值 → 标记执行偏差 → AI生成复盘 → 更新统计报表
```

## 6. 核心页面

| 页面 | 作用 |
|---|---|
| Dashboard | 账户状态、今日风险、机会摘要、系统状态 |
| Opportunity Radar | 自动机会雷达，展示候选交易计划 |
| Chart Workspace | 图表工作区，显示K线、结构、关键位、计划线 |
| Trade Plan Center | 交易计划中心，创建、检查、保存交易计划 |
| Order Preview | 订单预览与用户确认页 |
| Execution Monitor | 订单与持仓监控页 |
| Risk Center | 风控状态、禁交易原因、规则配置 |
| Journal | 交易日志 |
| Review | 复盘统计 |
| Settings | 交易所API、风控配置、AI配置、系统模式 |

## 7. 功能需求

## 7.1 行情与图表

- 支持多标的；
- 支持多周期；
- 支持K线图；
- 显示当前价格；
- 显示基础市场状态；
- 显示系统识别的关键位。

## 7.2 市场结构识别

- Swing High / Swing Low；
- 趋势识别；
- 震荡识别；
- BOS；
- CHOCH；
- 突破/假突破；
- 支撑压力区；
- 区间中部禁交易提示。

## 7.3 自动计划生成

系统根据市场结构生成候选计划：

- 趋势回踩；
- 区间边缘反弹；
- 突破回踩；
- 反抽失败；
- 假突破反打。

每个候选计划包含：

- symbol；
- direction；
- setupType；
- entryZone；
- stopLoss；
- targets；
- riskReward；
- grade；
- status；
- invalidationReason。

## 7.4 风控与仓位计算

- 动态风险金额计算；
- 以损定仓；
- 保证金计算；
- 杠杆限制；
- 最小盈亏比检查；
- 止损距离检查；
- 日亏损限制；
- 连亏限制；
- 冷却期；
- Kill Switch。

## 7.5 AI评估

AI输出必须基于算法结果和风控结果。

AI可以：

- 解释市场结构；
- 解释计划质量；
- 解释风险；
- 提示等待条件；
- 生成复盘总结。

AI不能：

- 绕过风控；
- 直接下单；
- 建议无止损交易；
- 建议超过系统上限的风险。

## 7.6 L4执行

- 生成订单预览；
- 用户确认后提交订单；
- 支持限价单优先；
- 支持逐仓优先；
- 支持止盈止损；
- 支持撤单；
- 支持订单状态同步；
- 支持执行日志。

---

## 8. 非功能需求

### 8.1 安全性

- API Key加密保存；
- 交易Key不允许提现权限；
- 支持IP白名单；
- 支持Kill Switch；
- 支持Dry Run；
- 敏感操作审计。

### 8.2 稳定性

- 交易所API异常时禁止新订单；
- WebSocket断开时禁止新订单；
- 止损设置失败时报警；
- 订单状态不一致时进入安全模式。

### 8.3 可扩展性

- 风控规则可配置；
- 配置有版本；
- 交易所适配器可替换；
- AI供应商可替换；
- 结构识别算法可迭代。

### 8.4 可观测性

- 所有风控决策记录；
- 所有订单请求记录；
- 所有交易所响应记录；
- 所有AI评估记录；
- 所有用户确认记录。
