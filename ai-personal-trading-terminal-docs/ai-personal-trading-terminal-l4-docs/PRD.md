# 产品需求文档 PRD

## 1. 产品名称

**AI Personal Trading Terminal L4**  
中文名：**AI个人交易终端 L4**

---

## 2. 产品定位

一个面向个人加密货币合约交易者的半自动交易工作台。

系统通过规则算法、AI评估和交易所API集成，帮助用户完成从市场分析到用户确认执行的完整交易流程。

一句话定位：

> 基于算法风控与AI评估的个人半自动交易终端。

---

## 3. 核心价值

### 3.1 减少人工计算错误

系统自动根据账户权益、风险比例、入场价、止损价、杠杆、交易所精度规则计算：

- 风险金额；
- 止损距离；
- 最大名义仓位；
- 合约数量；
- 所需保证金；
- 预估手续费；
- 盈亏比。

### 3.2 减少临场情绪干扰

系统在下单前强制进行：

- 风控检查；
- 机会评级；
- 订单预览；
- 用户确认；
- 纪律状态检查。

### 3.3 提升交易执行效率

系统自动生成交易所订单参数，避免用户在交易平台中重复填写：

- 方向；
- 价格；
- 数量；
- 杠杆；
- 止损；
- 止盈；
- 保证金模式。

### 3.4 形成复盘闭环

系统自动记录：

- 计划；
- 风控结果；
- AI评估；
- 用户确认；
- 下单请求；
- 交易所响应；
- 成交记录；
- 持仓变化；
- 平仓结果；
- 复盘标签。

---

## 4. 用户角色

### 4.1 个人交易者

核心用户，使用系统进行个人交易辅助。

需求：

- 快速了解市场结构；
- 自动生成候选计划；
- 自动计算仓位；
- 快速完成下单；
- 控制风险；
- 复盘交易表现。

### 4.2 系统管理员

本项目v1.0为单用户系统，管理员通常也是用户本人。

职责：

- 管理API Key；
- 管理风控配置；
- 开启/关闭执行模式；
- 查看系统日志；
- 处理异常。

---

## 5. 关键业务流程

## 5.1 自动机会发现流程

```text
行情同步 → 市场结构识别 → 关键位识别 → 候选计划生成 → 机会评级 → 风控初筛 → AI解释 → 机会雷达展示
```

## 5.2 手动计划检查流程

```text
用户填写计划 → 仓位计算 → 风控检查 → AI解释 → 保存计划 / 放弃计划 / 进入订单预览
```

## 5.3 L4确认执行流程

```text
计划通过风控 → 生成订单预览 → 用户二次确认 → Execution Engine提交订单 → 同步订单状态 → 设置/确认TP SL → 进入持仓监控 → 自动记录日志
```

## 5.4 复盘流程

```text
订单关闭 → 生成交易记录 → 计算R值 → 标记执行偏差 → AI生成复盘 → 更新统计报表
```

## 7. 功能需求

## 7.1 行情与图表

- 支持多标的；
- 支持多周期；
- 支持K线图；
- 显示当前价格；
- 显示基础市场状态；
- 显示系统识别的关键位。

## 7.2 市场结构识别

- Swing High / Swing Low；
- 趋势识别；
- 震荡识别；
- BOS；
- CHOCH；
- 突破/假突破；
- 支撑压力区；
- 区间中部禁交易提示。

## 7.3 自动计划生成

系统根据市场结构生成候选计划：

- 趋势回踩；
- 区间边缘反弹；
- 突破回踩；
- 反抽失败；
- 假突破反打。

每个候选计划包含：

- symbol；
- direction；
- setupType；
- entryZone；
- stopLoss；
- targets；
- riskReward；
- grade；
- status；
- invalidationReason。

## 7.4 风控与仓位计算

- 动态风险金额计算；
- 以损定仓；
- 保证金计算；
- 杠杆限制；
- 最小盈亏比检查；
- 止损距离检查；
- 日亏损限制；
- 连亏限制；
- 冷却期；
- Kill Switch。

## 7.5 AI评估

AI输出必须基于算法结果和风控结果。

AI可以：

- 解释市场结构；
- 解释计划质量；
- 解释风险；
- 提示等待条件；
- 生成复盘总结。

AI不能：

- 绕过风控；
- 直接下单；
- 建议无止损交易；
- 建议超过系统上限的风险。

## 7.6 L4执行

- 生成订单预览；
- 用户确认后提交订单；
- 支持限价单优先；
- 支持逐仓优先；
- 支持止盈止损；
- 支持撤单；
- 支持订单状态同步；
- 支持执行日志。

## 8. 非功能需求

> 本节为单用户个人交易终端的指标基线，非机构级 HFT 标准。指标按版本递进：v0.1/v0.2 仅需满足"可用"，v0.8 起需满足"可执行真实订单"，v1.0 需满足全部指标。

### 8.1 安全性

- API Key加密保存；
- 交易Key不允许提现权限；
- 支持IP白名单；
- 支持Kill Switch；
- 支持Dry Run；
- 敏感操作审计。

### 8.2 稳定性

- 交易所API异常时禁止新订单；
- WebSocket断开时禁止新订单；
- 止损设置失败时报警；
- 订单状态不一致时进入安全模式。

### 8.3 可扩展性

- 风控规则可配置；
- 配置有版本；
- 交易所适配器可替换；
- AI供应商可替换；
- 结构识别算法可迭代。

### 8.4 可观测性

- 所有风控决策记录；
- 所有订单请求记录；
- 所有交易所响应记录；
- 所有AI评估记录；
- 所有用户确认记录。

### 8.5 性能指标

| 指标 | 目标 | 适用版本 | 说明 |
|---|---|---|---|
| 只读 API P95 延迟 | ≤ 200 ms | v0.1+ | 计划列表、风控状态、持仓查询 |
| 写入 API P95 延迟 | ≤ 500 ms | v0.1+ | 创建计划、保存配置、用户确认 |
| 风控引擎单次检查延迟 | ≤ 50 ms | v0.1+ | 纯函数，无 IO，见 `MODULES.md §8` |
| 订单预览生成延迟 | ≤ 800 ms | v0.1+ | 含风控 + 仓位计算 + 精度处理 |
| 用户确认 → 交易所下单延迟 | ≤ 1 s | v0.8+ | 含二次风控 + clientOid 生成 + API 调用 |
| 行情 tick → 前端推送延迟 | ≤ 200 ms | v0.2+ | WebSocket 链路 |
| K线收盘 → 结构更新延迟 | ≤ 2 s | v0.2+ | 含结构识别算法 |
| AI 评估延迟 | ≤ 5 s | v0.5+ | 单次 LLM 调用，超时降级为 `None` |
| 前端首屏加载 | ≤ 2 s | v0.1+ | 本地部署，不计公网 RTT |

### 8.6 并发与容量

| 维度 | 目标 | 说明 |
|---|---|---|
| 并发用户 | 1 | 单用户系统，见 §4.2 |
| 订阅标的数 | ≤ 50 | 行情订阅 + 结构识别，超出需评估 |
| 并发 WebSocket 连接 | ≤ 4 | 主交易所行情 + 私有订单流 + 备用 |
| 行情 tick 吞吐 | ≥ 100 ticks/s/标的 | 单标的高频时段峰值 |
| 数据库连接池 | 5–10 | 单用户无需大池 |
| 后台任务并发 | ≤ 8 | 结构识别 + 风控扫描 + 对账 + AI 评估 |
| 配置版本累积 | 无上限 | 不可变历史，仅 `is_active=true` 唯一 |

### 8.7 数据容量与保留

| 数据 | 保留策略 | 1 年预估容量 |
|---|---|---|
| K线（1m） | 90 天热 + 1 年冷归档 | ~50 MB / 标的 |
| K线（5m 及以上） | 永久 | ~10 MB / 标的 |
| `order_intents` + `order_state_events` | 永久 | ~10 万行 / 年，<100 MB |
| `trade_journal` | 永久 | ~1 万行 / 年，<10 MB |
| `system_events` | 永久 | ~100 万行 / 年，~500 MB |
| `candidate_plans` | 30 天后归档 | 自动清理 `EXPIRED` 状态 |
| `review_reports` | 永久 | 单用户年增量可忽略 |
| 数据库总容量 | — | < 10 GB / 年（单用户） |

### 8.8 可用性与恢复

| 指标 | 目标 | 说明 |
|---|---|---|
| 交易时段可用性 | ≥ 99% | 个人终端，非 7×24 SLA；非交易时段可停机维护 |
| RTO（恢复时间目标） | ≤ 5 min | 手动重启 + 健康检查 |
| RPO（恢复点目标） | 0 | 订单提交前必须落库（`ORDER_LIFECYCLE.md §3.4`），不允许内存态订单 |
| 数据库备份频率 | 每日 | 见 `OPERATIONS.md §1` |
| 备份恢复测试 | 每季度 | 验证备份可还原 |

### 8.9 兼容性

| 维度 | 目标 |
|---|---|
| 浏览器 | Chromium 内核最新 2 个大版本（Chrome / Edge） |
| 操作系统（部署侧） | Linux x86_64（Ubuntu 22.04+ / Debian 12+） |
| 运行时 | Python 3.11+ / Node 20+（前端构建） |
| 数据库 | PostgreSQL 15+ |
| 显示分辨率 | ≥ 1366×768，推荐 1920×1080 |

### 8.10 降级策略

| 故障 | 降级行为 | 恢复条件 |
|---|---|---|
| 交易所 WebSocket 断开 | 禁止新单 + REST 轮询补数据 | 重连成功 + 对账通过 |
| AI 评估超时/不可用 | `ai_evaluation = None`，DecisionGate 按纯风控映射 | AI 服务恢复 |
| 数据库写入失败 | 禁止执行 + 保留内存上下文 | 连接恢复后补写事件 |
| 交易所 API 限流 | 排队 + 退避重试（非订单类） | 限流窗口结束 |
| 主加密密钥解密失败 | 禁止读取该 Key + 报警 | 见 `SECURITY.md §2.4.2` 异常处理 |
