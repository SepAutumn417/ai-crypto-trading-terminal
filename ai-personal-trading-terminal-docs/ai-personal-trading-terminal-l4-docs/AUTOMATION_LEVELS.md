# 自动化等级定义

## 1. 等级总览

| 等级 | 名称 | 描述 | 是否提交订单 |
|---|---|---|---|
| L0 | 手动记录 | 用户手动分析和下单，系统只记录 | 否 |
| L1 | 手动计划计算 | 用户输入计划，系统计算仓位 | 否 |
| L2 | 自动评估 | 系统自动识别结构和风险 | 否 |
| L3 | 自动生成计划 | 系统自动生成候选交易计划 | 否 |
| L4 | 用户确认后执行 | 系统准备订单，用户确认后提交交易所 | 是，必须确认 |
| L5 | 全自动交易 | 系统无需确认自动开平仓 | 是，v1不做 |

---

## 2. 当前目标：L4

本项目 v1.0 的自动化目标是 L4。

L4包含：

- 自动结构识别；
- 自动计划生成；
- 自动仓位计算；
- 自动风控检查；
- AI解释评估；
- 订单预览；
- 用户确认；
- 系统提交订单；
- 订单/持仓监控；
- 日志与复盘。

---

## 3. L4 与 L5 的边界

| 能力 | L4 | L5 |
|---|---|---|
| 自动生成计划 | 是 | 是 |
| 自动计算仓位 | 是 | 是 |
| 自动评估风险 | 是 | 是 |
| 用户确认 | 必须 | 不需要 |
| 自动开仓 | 否，需确认 | 是 |
| 自动反手 | 不允许 | 可能允许 |
| 自动加仓 | v1不允许 | 可能允许 |
| 风控绕过 | 不允许 | 不允许 |

---

## 4. L4 开启条件

系统只有满足以下条件才允许进入 L4：

1. Dry Run 通过；
2. 交易所只读同步稳定；
3. 订单参数生成测试通过；
4. 风控引擎测试通过；
5. Kill Switch 可用；
6. API Key权限配置正确；
7. 用户已启用 Execution Mode；
8. 当前无风控禁止状态。

---

## 5. L4 自动降级条件

出现以下情况，系统必须从 L4 降级到 L3 或只读模式：

- 交易所API错误率过高；
- WebSocket断开；
- 订单状态无法确认；
- 止损设置失败；
- 当日亏损达到限制；
- Kill Switch触发；
- API Key权限异常；
- 系统时间偏差过大；
- 数据库写入失败；
- 风控配置异常。

---

## 6. L5 暂不做的原因

- 当前系统目标是个人风险控制和执行效率；
- L5会显著增加策略风险；
- L5需要更复杂的回测、仿真、监控和风控；
- L5会放大模型误判和数据异常风险；
- L5可能使用户失去最终确认环节。

因此 v1.0 不做 L5。
